# Repository Instructions

## Scope

- This repo is an OpenCode skill workspace for the `rackula` skill.
- The skill source is `.agents/skills/rackula/SKILL.md`; update it when changing Rackula behavior or schema.
- Eval prompts live in `.agents/skills/rackula/evals/evals.json`; keep them aligned with skill changes.

## Commands

- `pnpm run build` — Compile TypeScript entrypoints in `src/` → `.agents/skills/rackula/scripts/*.js`
- `pnpm run zip-yaml -- --input <dir> --output <dir>` — Package YAML to ZIP
- Run `build` before `zip-yaml` if you modified source or are unsure
- See [docs/COMMANDS.md](./docs/COMMANDS.md) for full usage

## Generated Skill Scripts

- Do not manually edit `.agents/skills/rackula/scripts/*.js`; those files are generated artifacts bundled with the skill.
- Modify TypeScript sources in `src/*.ts` instead, then run `pnpm run build` to regenerate the skill scripts.
- If behavior changes in a bundled script, commit both the `src/*.ts` source change and the regenerated `.agents/skills/rackula/scripts/*.js` output.
- Keep `.agents/skills/rackula/scripts/package.json` as hand-authored runtime metadata; it is not generated by `build`.

## Rackula Layout Workflow

1. Create/edit `input/*.rackula.yaml` with `metadata.id` and `metadata.name`
2. Run `pnpm run build` then `pnpm run zip-yaml -- --input ./input --output ./output`
3. Report both YAML path and ZIP path after packaging
4. See [docs/COMMANDS.md](./docs/COMMANDS.md#workflow) for detailed steps

## Common YAML Mistakes

- Use `racks: [...]` (modern), not `rack:` (legacy)
- Positions are in `1/6U` units: U25 = `position: 150`; never `position: 0` except container children
- Top-level order: `metadata`, `version`, `name`, `racks`, `device_types`, `settings`, then optional `rack_groups`, `connections`, `cables`
- Define every `device_type` slug used in `racks[].devices[].device_type`
- Use `u_height >= 0.5`, hex colors `#RRGGBB`, rack widths `10`, `19`, `21`, or `23`
- Use `connections` for port-to-port links; `cables` is deprecated
- For blade/container: use `slots` on container + `container_id`/`slot_id` on children
- Quote `device_bays[].position` values: `position: "1"`
- Don't use `management` as interface `type`; use `1000base-t`, `10gbase-x-sfpp`, `console`, `virtual`, or `other`
- Preserve unknown fields when editing legacy YAML

## Troubleshooting

- pnpm does not pass positional args: use `pnpm run zip-yaml -- --input ./input --output ./output`, NOT `pnpm run zip-yaml -- ./input ./output`
- Bun build needs `--target=node` or `parseArgs` fails
- Missing `metadata.name` → error; `metadata.id` is optional

See [docs/TROUBLESHOOTING.md](./docs/TROUBLESHOOTING.md) for workarounds.
